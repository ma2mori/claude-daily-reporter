# 日報分析コマンド

以下のユーザーとの会話ログから、開発者の1日の活動を分析し、テンプレートタイプに応じた形式で出力してください。

## テンプレートタイプ: {TEMPLATE_TYPE}

## 分析対象のメッセージ:
{USER_MESSAGES}

## 出力形式:

{TEMPLATE_TYPE}がsimpleの場合：

## 今日やったこと
- 作業内容1(プロジェクト名)
- 作業内容2(プロジェクト名)
- 作業内容3(プロジェクト名)
- ...

{TEMPLATE_TYPE}がdetailの場合：

### 今日やったこと
- 実際に行った作業内容1
- 実際に行った作業内容2
- 実際に行った作業内容3

### 学んだこと
- 新しく学んだ技術や知識1
- 新しく学んだ技術や知識2
- 新しく学んだ技術や知識3

### 詰まったこと
- 技術的な問題や困難1
- 技術的な問題や困難2
- 技術的な問題や困難3

### 困ったこと
- 作業上の課題や悩み1
- 作業上の課題や悩み2
- 作業上の課題や悩み3

## 分析の指針:
- **今日やったこと**: 実装、修正、テスト、レビュー、デバッグなど具体的な作業
- 各項目は必ず末尾に(プロジェクト名)を付ける（simpleの場合のみ）
- プロジェクト名は[PROJECT:xxx]タグから抽出する
- 作業内容が多い場合は、10〜15項目程度まで箇条書きで記載
- 同じプロジェクトの作業でも、異なる作業内容であれば別項目として記載
- 開発に関連する内容のみを抽出
- できるだけ具体的に記載（例：「調査」ではなく「日報作成機能の不具合調査」）
- 重要度の高い作業を優先して記載
- 各プロジェクトタグ内のメッセージから、そのプロジェクトで行った作業を抽出する

## detailテンプレート用の追加指針:
- **学んだこと**: 新しい技術・知識・概念の習得、発見、理解
- **詰まったこと**: エラー、バグ、技術的な問題、実装の困難
- **困ったこと**: 理解の困難、方法論の悩み、作業上の課題
- 各項目は60文字以内で簡潔に要約（detailの場合）
- 該当する内容がない場合は「特になし」と記載（detailの場合）

## ユーザーメッセージから作業を推測する際のヒント:
- 「.gitignoreを現在のプロジェクト内を踏まえて最適なものに更新」→ .gitignoreファイルの最適化と更新
- 「`templates/*.md`について以下のようにする」→ テンプレートファイルの変更・再構成
- 「既に`reports/daily-report-yyyy-mm-dd.md`があり、同じ日付で作成された時、上書きするようにする」→ 既存ファイル上書き機能の実装
- 「〜を確認すると何も記載されない」→ 問題の調査と修正
- 実装・修正・改善・追加・削除・更新・変更などの動詞に注目
- ファイル名やディレクトリ名が言及されている場合は、それらに関する作業として記載

## アシスタントの作業メッセージから作業を抽出する際のヒント:
[ASSISTANT WORK]セクションには実際に行った作業の詳細が含まれます：
- 「ファイルを作成しました」→ 新規ファイルの作成
- 「ファイルを更新しました」→ 既存ファイルの修正・更新
- 「削除しました」→ ファイルやディレクトリの削除
- 「リネームしました」→ ファイル名の変更
- 「追加しました」→ 機能や設定の追加
- 「修正しました」→ バグ修正や問題の解決
- 「実装しました」→ 新機能の実装
- 具体的なファイル名（例：.gitignore、analyze-daily-work-simple.md）に注目
- 機能の説明（例：「プロジェクト名付き出力対応」「既存ファイル上書き通知」）を含める

## 作業内容の詳細度について:
- できるだけ具体的に記載する
- 何を、どのように変更したかを明確にする
- 追加した機能や修正内容の詳細を括弧内に含める
- 例：「日報テンプレートの再構成（simple.mdをデフォルトに、default.md→detail.md、reflection.md削除）」

## ツール結果から作業を抽出する際のヒント:
[TOOL RESULTS]セクションには実際のファイル操作結果が含まれます：
- 「File created successfully at: /path/to/file」→ 新規ファイルの作成
- 「The file /path/to/file has been updated」→ 既存ファイルの更新
- 「Deleted」「removed」→ ファイルやディレクトリの削除
- 「Renamed」「moved」→ ファイル名の変更
- ファイルパスから具体的なファイル名を抽出して作業内容に含める
